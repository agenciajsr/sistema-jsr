  <action>
    1. Criar `src/lib/validations/usuario.ts`:
    ```typescript
    import { z } from 'zod'

    export const usuarioSchema = z.object({
      nome: z.string().min(1, 'Nome é obrigatório'),
      email: z.string().email('Email inválido'),
      senhaTemporaria: z.string().min(8, 'Senha deve ter ao menos 8 caracteres'),
      role: z.enum(['admin', 'membro']),
    })

    export type UsuarioInput = z.infer<typeof usuarioSchema>
    ```

    2. Criar `src/actions/usuarios.ts` com a Server Action `criarUsuario(input: UsuarioInput)`:
       - Primeiro chama `requireAdmin()` de `@/lib/auth/session` — se retornar `{ error }`, devolve `{ error }` imediatamente sem prosseguir (D-02: apenas Admin cria usuários).
       - Valida `input` com `usuarioSchema.safeParse`.
       - Cria um cliente Supabase Admin (`createClient` de `@supabase/supabase-js` com `SUPABASE_SECRET_KEY`, nunca `NEXT_PUBLIC_`).
       - Chama `supabaseAdmin.auth.admin.createUser({ email: input.email, password: input.senhaTemporaria, email_confirm: true })`.
       - Se sucesso, insere em `profiles` via `db.insert(profiles).values({ id: data.user.id, nome: input.nome, role: input.role })`.
       - Retorna `{ data: { id, email } }` em caso de sucesso, `{ error: string }` em caso de falha (usar a mensagem padrão "Não foi possível salvar. Verifique os dados e tente novamente." do UI-SPEC para erros inesperados).
  </action>
