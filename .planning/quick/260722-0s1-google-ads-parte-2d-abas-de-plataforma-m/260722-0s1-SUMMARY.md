# Quick 260722-0s1: Google Ads Parte 2d — abas de plataforma Meta/Google/Compilado Summary

Adiciona abas [Meta] [Google] [Compilado] no painel de um cliente em `/campanhas`, controladas pelo searchParam `?plataforma=` (mesmo padrão server-render do `?periodo=`). Reutiliza 100% dos componentes de exibição — só troca a fonte de dados por plataforma e adiciona um seletor de abas + um badge. Cliente com uma só plataforma continua idêntico a hoje; a aba Google mostra só o que a Parte 2b grava, ocultando graciosamente o resto.

## What Was Built

### Task 1 — Camada de dados plataforma-aware (commit d8c71be)
`src/lib/trafego/painel.ts` + `src/lib/trafego/aggregate.ts`:

- **`getPainelCampanhas(clienteId, periodo, plataforma?)`** — 3º parâmetro opcional. O predicado `eq(adAccounts.plataforma, plataforma)` entra no `and(...)` **só quando `plataforma` está definido** (spread condicional). Ausente = todas as contas = comportamento idêntico ao de hoje. `contasUnificadas` passa a refletir só as contas da plataforma filtrada por aba.
- **`getInvestido30dPorCliente()`** — passa a devolver `Map<string, { meta: number; google: number }>`. Query ganha `plataforma: adAccounts.plataforma` e `.groupBy(clienteId, plataforma)`; monta mapa aninhado. O total (meta+google) preserva o número exibido hoje quando só há Meta.
- **`getPlataformasDoCliente(clienteId)`** — nova query `selectDistinct` das plataformas com conta ativa. Decide se o cliente tem 1 ou 2 plataformas (base em CONTAS, não em gasto).

### Task 2 — Seletor de abas + badge (commit 7bab06b)
`src/components/trafego/seletor-plataforma.tsx` (novo, 65 linhas), exporta 2 itens:

- **`SeletorPlataforma`** — abas Meta/Google/Compilado via `Tabs`/`TabsList`/`TabsTrigger`. `trocarPlataforma` monta `URLSearchParams` preservando `cliente` e `periodo` lidos FRESCOS de `useSearchParams()` e faz `router.push('/campanhas?...')` — espelha `SeletorCampanhas`.
- **`BadgePlataforma`** — selinho discreto com tom por plataforma (Meta azul/primary, Google âmbar/chart-warning, Compilado neutro/muted).

### Task 3 — Integração na página + soloCampanhas + split nos cards (commit 744b596)
- **`campanhas/page.tsx`**: resolve `plataformasCliente` ANTES do painel; `temAbas = length >= 2`; `abaAtiva` (default Meta) só com 2 plataformas; `plataformaFiltro` ('compilado'/1-plataforma → undefined); passa o filtro como 3º arg a `getPainelCampanhas`. Cabeçalho ganha `<BadgePlataforma>` (sempre) e `<SeletorPlataforma>` (só com abas). Aba Google (`ehGoogle`): `TabelaNiveis soloCampanhas`, oculta Demografia/Regiões/Criativos (`{!ehGoogle && ...}`) e mostra uma nota "em breve para Google Ads". KPIs/Gráfico/Tabela/Funil/Ações do dia permanecem nas 3 abas.
- **`tabela-niveis.tsx`**: prop `soloCampanhas?: boolean` — quando true, oculta os TabsTrigger Conjuntos/Anúncios (trava no nível campanhas).
- **`landing-clientes.tsx`**: `investido30d` agora é `Map<string, {meta,google}>`; card usa `total = meta+google`; linha "Meta R$X · Google R$Y" só quando `inv.google > 0` (cliente só-Meta sem a linha = visual idêntico a hoje).

## REGRA DURA — nenhum número Meta muda
- Cliente de 1 plataforma → `getPlataformasDoCliente` retorna 1 → `temAbas=false` → `abaAtiva=null` → `plataformaFiltro=undefined` → `getPainelCampanhas` seleciona as únicas contas que existem = **byte-a-byte idêntico ao de hoje**.
- Aba Compilado (cliente de 2 plataformas) → `plataformaFiltro=undefined` → soma meta+google (comportamento unificado da Parte 2c).
- Aba Meta → `plataformaFiltro='meta'` → só contas 'meta', números iguais ao painel Meta-only.
- Nenhuma função pura foi tocada; vitest permanece 649/649.
