# Quick 260714-j92: Clientes — Lista Compacta com Abas por Status Summary

Página /clientes trocada de cards grandes com N+1 (1 query por cliente) para uma lista compacta de 1 linha/cliente, alimentada por 6 queries agregadas sequenciais, com busca, abas por status e os dois status novos que abrem espaço para o CRM.

## O que foi feito

**Task 1 — Status novos** (`ea55dcb`): enum `cliente_status` estendido com `aguardando_inicio` e `em_aviso` (valores no fim, pois `ALTER TYPE ADD VALUE` só acrescenta). Migration `0014_futuristic_bedlam.sql` **gerada** contendo exatamente 2 `ADD VALUE` e nada mais. Zod, `ClienteFormEditar` e os dois `Record<Cliente['status'], string>` exaustivos atualizados.

**Task 2 — Dados sem N+1** (`4dd1991` RED, `83856ac` GREEN): `agregar.ts` (módulo puro, zero import de db/react) + 14 testes vitest; `lista.ts` com `getClientesLista()` fazendo 6 queries agregadas **sequenciais** e devolvendo `[]` em erro.

**Task 3 — UI** (`773d120`): página server enxuta + `clientes-lista.tsx` (busca, 6 abas com contador, tabela compacta, linha clicável, dois estados vazios). `cliente-card.tsx` removido.

## Decisões

**1. Os status novos são ADITIVOS.** As consultas que filtram `status='ativo'` (dashboard, financeiro, relatórios, alertas, copilot) **não foram alteradas**. Mudar agora a semântica de "cliente ativo" — por exemplo, passar a contar `em_aviso` como ativo no MRR — alteraria números de outras telas sem o usuário ter pedido. Fica como decisão explícita a revisitar quando o CRM entrar. Confirmado por `git diff --stat master`: nenhum arquivo de `src/lib/alertas/`, `src/lib/ai/copilot.ts`, dashboard, financeiro ou relatórios aparece no diff.

**2. `lista.ts` não leva `'use server'`.** É consumido direto pelo Server Component da página; marcar como Server Action exporia um endpoint sem necessidade.

**3. Tabela única abaixo do `TabsList`.** A lista já chega filtrada por `aba`, então duplicá-la em 6 `TabsContent` só multiplicaria DOM.

**4. `pausado` e `em_aviso` compartilham o âmbar (#D97706) no `STATUS_COLOR` da ficha.** A distinção visual entre os dois vive no badge da lista (`bg-muted` vs `bg-chart-warning`), onde os dois status aparecem lado a lado.

## Padrão reforçado

Função de dados = **poucas queries agregadas sequenciais + merge em memória via módulo puro testado**. O pool é `max=3` com `max_pipeline=0`: o Supavisor em transaction mode não suporta pipelining, então queries paralelas na mesma conexão penduram para sempre (causa raiz dos travamentos dos quicks 260713-usi e 260714-ita). Mesmo padrão do 260714-ita. Guards no código: `grep "Promise.all" src/lib/clientes/lista.ts` e `grep "new Date()" src/lib/clientes/lista.ts` voltam vazios — inclusive nos comentários, que foram reescritos para não gerar falso positivo na verificação automatizada.

## Pendências

As abas correspondentes simplesmente aparecem com contador 0 — a leitura não quebra.
