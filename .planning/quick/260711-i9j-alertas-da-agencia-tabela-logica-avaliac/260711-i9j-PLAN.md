   - titulo "Pagamento vencido", detalhe "R$ X — {descricao}" (com nome do cliente se tiver)

   - Filtra status === 'pausado' ou status === 'encerrado'
   - severidade: info
   - titulo "Cliente {status}" (pausado/encerrado)
   - detalhe "Sem atividade — verificar situacao"
   - dataRelevante = data atual (ISO string)

Exportar funcao `ordenarPorSeveridade(alertas: Alerta[]): Alerta[]` que ordena usando SEVERIDADE_ORDEM.

**src/actions/alertas.ts** — Server Action seguindo padrao de financeiro.ts:
- `'use server'` no topo
- Importar `getCurrentUser` de `@/lib/auth/session`
- Importar `db` de `@/lib/db`, tabelas `contratos, transacoes, clientes` do schema
- Importar `avaliarContratos, avaliarTransacoes, avaliarClientesInativos, ordenarPorSeveridade` de `@/lib/alertas/avaliar`
- Importar `eq, and, lte, gte, sql` de `drizzle-orm`
- `export async function getAlertas(): Promise<Alerta[]>`
  1. Verifica sessao com `getCurrentUser()`, retorna [] se nao logado
  2. Calcula `hoje` como YYYY-MM-DD e `limite30d` como hoje + 30 dias (date-fns `addDays` + `format`)
  3. Query contratos a vencer: select contratos JOIN clientes ON clienteId, where dataVencimento <= limite30d (inclui ja vencidos, sem lower bound — mostra todos vencidos). Campos: id contrato, clienteId, clienteNome (clientes.nome), dataVencimento, valorMensal
  4. Query transacoes vencidas: select transacoes LEFT JOIN clientes, where status = 'vencido'. Campos: id, clienteId, clienteNome (ou null), descricao, valor, data, status
  5. Query clientes inativos: select clientes where status IN ('pausado', 'encerrado'). Campos: id, nome, status
  6. Chama as 3 funcoes de avaliacao com os resultados das queries
  7. Concatena os 3 arrays e ordena com `ordenarPorSeveridade`
  8. Retorna o array unificado
