**3. src/actions/clientes.ts** — Duas mudancas:

a) `createClienteComContrato` e `updateCliente` ja funcionam genericamente porque fazem `parsed.data` no insert/set. Como o schema agora tem mais campos, o Drizzle vai inserir/atualizar os novos campos automaticamente. Porem, para campos numericos que viram `undefined` apos transform, converter para `null` antes do insert/update para que o Drizzle envie NULL ao Postgres (e nao omita o campo). Criar um helper:

```typescript
function clienteParaDb(data: ClienteInput) {
  return {
    ...data,
    diaPagamento: data.diaPagamento ?? null,
    gestorId: data.gestorId ?? null,
    verbaMensal: data.verbaMensal != null ? String(data.verbaMensal) : null,
    ticketMedio: data.ticketMedio != null ? String(data.ticketMedio) : null,
    servicosContratados: data.servicosContratados ?? [],
  }
}
```

Usar `clienteParaDb(parsed.data)` no `.values()` do insert e no `.set()` do update (em vez de `parsed.data` direto). Campos numeric do Drizzle esperam string (nao number) — converter verbaMensal/ticketMedio para String().
