## O que foi feito

### Schema (schema.ts)
- 2 enums: `tipoPessoaEnum` (fisica/juridica), `formaPagamentoEnum` (pix/boleto/cartao/transferencia)
- ~20 colunas novas na tabela `clientes`: dados fiscais, endereco, online, pagamento, servicos (JSONB), operacao (gestor FK, verba, ticket medio, agendamento), observacoes
- Relacao `gestor` em `clientesRelations` (one → profiles)

### Validacao (cliente.ts)
- `clienteSchema` expandido com todos os campos
- Transforms para campos que chegam como string vazia do form (diaPagamento, gestorId, verbaMensal, ticketMedio)
- `SERVICOS_DISPONIVEIS` exportado como const array

### Actions (clientes.ts)
- Helper `clienteParaDb()` converte ClienteInput para formato Drizzle (numeric → string, undefined → null)
- `createClienteComContrato` e `updateCliente` usam o helper
- Nova action `getProfiles()` retorna id+nome dos membros

### Formulario (cliente-form.tsx)
- 7 Cards visuais: Dados do Cliente, Contato, Endereco, Contrato/Pagamento, Servicos Contratados, Operacao, Observacoes
- Modo criar: 7 secoes (inclui contrato)
- Modo editar: 7 secoes (Pagamento em vez de Contrato, sem datas/valor)
- Select de estado com 27 UFs brasileiras
- Checkboxes de servicos com Controller + array toggle
- Select de gestor carregado via useEffect + getProfiles()
- Campos condicionais: razaoSocial (PJ), frequenciaPosts (agendamento ativo)
- Placeholder dinamico no documento (CPF/CNPJ conforme tipoPessoa)
