**3. Server Actions (src/actions/financeiro.ts):**

a) Atualizar `createTransacao`: incluir os novos campos no insert (centroCusto, recorrencia, formaPagamento, responsavelId). Se recorrencia !== 'avulsa', chamar `gerarParcelasRecorrentes` automaticamente apos criar a transacao pai.

b) Nova action `gerarParcelasRecorrentes(transacaoPaiId: string, input: TransacaoInput, contratoVencimento?: string)`:
- Se `input.clienteId` existe: buscar contrato vigente do cliente (dataInicio <= hoje AND dataVencimento >= hoje). Usar `contratoVencimento` como data final. Se nao tiver contrato, gerar 12 meses.
- Se nao tem clienteId (despesa da agencia): gerar 12 meses.
- Calcular numero de parcelas: para mensal = meses entre hoje e data final; para trimestral = trimestres entre hoje e data final.
- Inserir todas as parcelas de uma vez com `db.insert(transacoes).values([...])`, cada uma com `transacaoPaiId` apontando para a transacao pai, `data` incrementada por 1 mes (ou 3 meses para trimestral), status 'pendente'.
- Usar `date-fns` (addMonths) para calculo de datas.

c) Nova action `getPrevisaoCaixa()`:
- Periodo: proximos 30 dias (hoje ate hoje + 30).
- Receitas previstas: transacoes tipo='receita' com status='pendente' e data dentro do periodo + soma de valorMensal de contratos vigentes que nao tenham transacao pendente correspondente no periodo.
- Despesas previstas: transacoes tipo='despesa' com status='pendente' e data dentro do periodo.
- Media historica: para despesas variaveis (categoria='outro'), calcular media dos ultimos 3 meses.
- Retornar: `{ totalReceber: number, totalPagar: number, saldoProjetado: number, items: Array<{ descricao, valor, data, tipo }> }`.

d) Nova action `getContasAReceber()`: buscar transacoes com tipo='receita' e status IN ('pendente', 'vencido'), leftJoin com clientes para nome, ordenar por data ASC. Incluir campos novos no select (centroCusto, formaPagamento, responsavelId, comprovanteUrl). Tambem fazer leftJoin com profiles para nome do responsavel.

e) Nova action `getContasAPagar()`: igual a `getContasAReceber` mas com tipo='despesa'.

f) Nova action `uploadComprovante(transacaoId: string, formData: FormData)`:
- Extrair arquivo do FormData.
- Usar `createClient` do Supabase (server-side com service role key) para upload no bucket 'comprovantes' com path `${transacaoId}/${filename}`.
- Atualizar campo `comprovanteUrl` na transacao com a URL publica.
- Retornar `{ data: { url: string } }` ou `{ error: string }`.

g) Atualizar `listTransacoes`: incluir novos campos no select (centroCusto, formaPagamento, responsavelId, comprovanteUrl). Fazer leftJoin com profiles para obter nome do responsavel.
